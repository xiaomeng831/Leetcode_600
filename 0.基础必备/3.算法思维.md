
# LeetCode 思维总结

LeetCode 本质不是背题，也不是背模板，而是在训练你把一个问题转化成：

状态、边界、选择、关系、转移

也就是：

状态：我现在知道什么？
边界：我处理到哪里了？
选择：我下一步能做什么？
关系：元素之间有什么依赖？
转移：旧状态怎么推出新状态？

---

## 1. 扫描状态思维

### 代表题型

```text
字符串
数组
模拟题
```

### 核心思想

从左到右扫描，每读一个元素，就更新当前状态。

你要不断问自己：

```text
我现在读到哪里了？
当前这一段是什么状态？
什么时候结算？
结算后状态怎么重置？
```

### 常见状态变量

```text
count
currentChar
max
start
end
seen
```

### 典型题目

```text
Count and Say
压缩字符串
反转单词
Zigzag Conversion
```

### 底层理解

很多字符串题本质都是：

```text
一边扫描，一边维护当前段的信息。
遇到边界时结算答案，然后重置状态。
```

---

## 2. 指针边界思维

### 代表题型

```text
双指针
快慢指针
二分查找
```

### 核心思想

重点不是“用了两个指针”，而是：

```text
每个指针代表什么边界？
为什么这个指针可以移动？
移动之后会不会漏掉答案？
```

### 常见变量

```text
left
right
mid
slow
fast
```

### 双指针最重要的问题

```text
我移动这个指针，为什么不会错过正确答案？
```

### 例子：Container With Most Water

盛水最多的容器里，为什么移动短板？

因为面积由短板决定。

```text
如果移动长板，短板不变，宽度变小，面积不可能变大。
所以只能移动短板，才有机会找到更大的面积。
```

### 底层理解

指针题的核心是边界收缩：

```text
每次移动一个边界，都要能解释为什么被丢掉的部分不可能是答案。
```

## 3. 区间窗口思维

### 代表题型

```text
滑动窗口
前缀和
差分数组
```

### 核心思想

把问题转成：

```text
我现在维护的是哪一段区间？
```

---

### 滑动窗口要问的问题

```text
右边什么时候进？
左边什么时候出？
当前窗口满足什么条件？
答案什么时候更新？
```

### 前缀和要问的问题

```text
区间和能不能用两个前缀状态相减得到？
```

核心公式：

```text
sum[i...j] = prefix[j + 1] - prefix[i]
```

### 差分数组要问的问题

```text
区间更新能不能只记录开始和结束的位置？
```

核心公式：

```text
diff[l] += val
diff[r + 1] -= val
```

### 底层理解

很多子数组题不是暴力枚举所有区间，而是维护区间状态。

```text
暴力枚举区间：O(n²)
维护区间状态：通常可以优化到 O(n)
```

---

## 4. 记忆历史思维

### 代表题型

```text
HashMap
HashSet
数组计数
缓存
```

### 核心思想

问自己：

```text
我需要记住过去发生过什么？
```

### 典型例子

#### Two Sum

```text
map 记录之前出现过的数。
当前数只需要查 complement 是否出现过。
```

#### 无重复字符的最长子串

```text
set / map 记录当前窗口里出现过的字符。
```

#### 前缀和为 K

```text
map 记录某个 prefixSum 出现过几次。
```

### 底层理解

HashMap / HashSet 的价值是：

```text
用空间换时间。
把“回头查历史”的操作从 O(n) 降到 O(1)。
```

所以很多题可以从：

```text
O(n²)
```

优化成：

```text
O(n)
```

## 5. 容器语义思维

### 代表题型

```text
栈
队列
堆
单调栈
优先队列
```

### 核心思想

不要死记“这题用栈”或者“这题用队列”。

而是问：

```text
这个容器帮我保存什么语义？
```

---

### 栈 Stack

语义：

```text
保存还没解决的问题。
后进先出。
```

典型题目：

```text
括号匹配
每日温度
单调栈
表达式计算
```

底层理解：

```text
栈适合处理“最近的、还没被匹配或解决的元素”。
```

---

### 队列 Queue

语义：

```text
保存下一批要处理的节点。
先进先出。
```

典型题目：

```text
BFS
层序遍历
最短路径
```

底层理解：

```text
队列适合一层一层地处理问题。
```

---

### 堆 Heap / PriorityQueue

语义：

```text
动态维护最大或最小的 K 个东西。
```

典型题目：

```text
Top K Frequent Elements
Kth Largest Element
Merge K Sorted Lists
```

底层理解：

```text
堆适合动态获取当前最大值或最小值。
```

---

## 6. 搜索遍历思维

### 代表题型

```text
DFS
BFS
图
树
矩阵搜索
```

### 核心思想

问自己：

```text
当前在哪个节点？
下一步能去哪？
有没有访问过？
什么时候停止？
```

---

### DFS

DFS 更像：

```text
一条路走到底。
```

适合：

```text
枚举所有可能
树的递归
路径搜索
连通块搜索
```

典型题目：

```text
二叉树路径
岛屿数量
组合问题
排列问题
```

---

### BFS

BFS 更像：

```text
一层一层扩散。
```

适合：

```text
最短路径
层序遍历
最少步数
```

典型题目：

```text
二叉树层序遍历
迷宫最短路径
单词接龙
腐烂橘子
```

---

### 底层理解

```text
最短路径，多数用 BFS。
枚举所有可能，多数用 DFS / 回溯。
树的递归，多数是 DFS。
```

典型题型：

```text
岛屿数量
迷宫
二叉树路径
课程表
图遍历
```

---

## 7. 递归分解思维

### 代表题型

```text
树
链表
分治
递归
```

### 核心思想

问自己：

```text
当前函数只负责什么？
子问题返回什么？
base case 是什么？
```

---

### 例子：反转链表

递归思维不是想着一次性反转整个链表，而是：

```text
我只处理当前节点和后面已经反转好的部分之间的关系。
```

---

### 例子：二叉树

树的问题尤其明显：

```text
左子树给我什么？
右子树给我什么？
我这一层怎么合并？
```

### 底层理解

递归训练的是：

```text
相信子问题已经解决。
```

你不需要在脑子里展开所有递归层。

你只需要定义清楚：

```text
当前函数的职责
子问题的返回值
当前层如何合并结果
递归什么时候结束
```

---

## 8. 选择撤销思维

### 代表题型

```text
回溯
组合
排列
子集
括号生成
N 皇后
```

### 核心思想

回溯的本质是：

```text
我现在做一个选择。
递归进入下一层。
回来后撤销这个选择。
```

### 标准模板

```java
path.add(x);
backtrack(...);
path.remove(path.size() - 1);
```

### 典型题目

```text
Subsets
Permutations
Combinations
Generate Parentheses
N-Queens
```

### 底层理解

回溯和普通 DFS 的区别是：

```text
DFS 只是遍历。
回溯是在遍历“选择树”。
```

每一层都要问：

```text
我现在有哪些选择？
这个选择是否合法？
选择之后进入哪里？
回来之后怎么撤销？
```

---

## 9. 状态转移思维

### 代表题型

```text
动态规划
```

### 核心思想

DP 不是背公式，而是问：

```text
dp[i] 表示什么？
它从哪些旧状态来？
初始状态是什么？
遍历顺序是什么？
```

---

### 例子：爬楼梯

状态定义：

```text
dp[i] = 到第 i 阶的方法数
```

状态转移：

```text
dp[i] = dp[i - 1] + dp[i - 2]
```

---

### 例子：打家劫舍

状态定义：

```text
dp[i] = 偷到第 i 家时的最大金额
```

状态转移：

```text
dp[i] = max(dp[i - 1], dp[i - 2] + nums[i])
```

---

### 例子：零钱兑换

状态定义：

```text
dp[i] = 凑出金额 i 的最少硬币数
```

状态转移：

```text
dp[i] = min(dp[i], dp[i - coin] + 1)
```

### 底层理解

DP 最难的不是代码，而是状态定义。

只要状态定义清楚，后面通常就是：

```text
旧状态 -> 新状态
```

---

## 10. 局部最优与反例思维

### 代表题型

```text
贪心
```

### 核心思想

贪心不是“感觉每次选最好的”。

而是要问：

```text
我现在这个选择，会不会影响后面？
有没有反例？
能不能证明局部最优推出全局最优？
```

---

### 典型例子

#### 区间问题

常见策略：

```text
按结束时间排序。
```

因为：

```text
越早结束，后面能留出越多空间。
```

---

#### 跳跃游戏

核心状态：

```text
维护当前能到达的最远位置。
```

---

#### 买卖股票

核心状态：

```text
维护目前见过的最低价格。
用当前价格减最低价格，更新最大利润。
```

### 底层理解

贪心题最怕的是：

```text
看起来能贪，其实不能贪。
```

所以每次想用贪心，都要尝试问：

```text
有没有反例？
这个局部选择为什么不会破坏全局最优？
```

# 做题时最实用的 6 个问题

真正做题时，不要死背一堆名词。

脑子里重点问这几个问题：

```text
1. 这是线性扫描、区间、搜索，还是状态转移？

2. 我要维护什么状态？

3. 这个状态用什么数据结构装？

4. 指针 / 循环 / 递归的边界是什么？

5. 每一步怎么保证不漏答案？

6. 最后答案在什么时候更新？
```

# 最终压缩版

```text
扫描状态思维：
从左到右扫，每一步更新当前状态。

指针边界思维：
每个指针代表一个边界，移动指针时要保证不漏答案。

区间窗口思维：
维护一段区间，思考右边何时进、左边何时出、答案何时更新。

记忆历史思维：
用 HashMap / HashSet 记住过去出现过的状态，把回头查变成 O(1)。

容器语义思维：
栈、队列、堆不是模板，而是保存不同语义的工具。

搜索遍历思维：
DFS 一条路走到底，BFS 一层一层扩散。

递归分解思维：
当前函数只解决当前层，子问题交给递归。

选择撤销思维：
做选择，递归，撤销选择，本质是遍历选择树。

状态转移思维：
定义 dp 状态，用旧状态推出新状态。

局部最优与反例思维：
贪心要证明局部最优能推出全局最优，并且没有反例。
```

# 补充：

#### 一、 算法维度的三个“边角料”补充
这 10 个思维覆盖了主流算法，但还有几类特定题型需要补充特定的思维模式，防止在面试中被冷门题目“偷袭”：

* **连通性思维 (并查集 Union-Find)：** 当你遇到“朋友圈”、“岛屿合并”、“判断图中是否有环”时，虽然可以用 DFS/BFS（搜索遍历思维）解决，但并查集专门解决动态连通性问题。它的核心是：这两个节点属于同一个集合吗？如何快速合并两个集合？
* **前缀树思维 (Trie)：** 当你处理海量字符串的前缀匹配、搜索提示（Auto-complete）时。它的核心是将字符串集合转化为树形状态图，用空间换取极速的字符级检索时间。
* **位运算思维 (Bit Manipulation)：** 常见于要求 $O(1)$ 空间复杂度的题目，或者状态压缩 DP。核心在于利用二进制的特性（如异或消除相同元素、位掩码记录集合状态）来极大地优化空间和常数级时间。

---

#### 二、 面试维度的“工程化”升华
对于高级后端开发岗位，面试官在观察你运用这 10 个思维写代码时，脑子里其实在评估另一套标准。你需要将算法思维与工程习惯结合：

* **Production-Ready (生产级) 代码思维：** 写出能 AC（通过）的代码是不够的。你的代码需要具备高可读性：变量命名是否清晰？是否将复杂的内部逻辑抽取成了私有方法（Helper Functions）？例如，在写 DFS 时，边界检查是否优雅？这是大厂极其看重的代码品味。
* **极限边界与防御性编程：** 你的“选择撤销思维”或“状态转移思维”是否考虑了输入为 `null`、空数组、或者大整数溢出（Integer Overflow）的情况？高级工程师需要在写核心逻辑前，先筑起防御底线。
* **系统上下文与扩展性思维：** 面试官经常会在你写完最优解后进行 Follow-up（追问）。例如，你用“记忆历史思维”写了一个基于 `HashMap` 的完美解法，面试官可能会问：“如果这个输入数据流极大，单机内存放不下怎么办？” 或者 “如果在多线程并发环境下调用这个方法，你的解法是线程安全的吗？” 这就要求你能跳出单机算法，结合底层的并发知识或分布式思维来回答。

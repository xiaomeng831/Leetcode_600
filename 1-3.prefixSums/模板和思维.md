# Prefix Sum / Difference Array 模板总结

## 1. 模板: 1D Basic Prefix Sum

适用场景: 频繁查询一维数组某个区间的和.

| 题目                          | 核心思路                                                                          |
| --------------------------- | ----------------------------------------------------------------------------- |
| Running Sum of 1d Array     | 从左到右累计当前总和                                                                    |
| Range Sum Query - Immutable | `prefix[i + 1] = prefix[i] + nums[i]`, `sum(l, r) = prefix[r + 1] - prefix[l]` |
| Find Pivot Index            | 判断 `leftSum == totalSum - leftSum - nums[i]`                                  |

核心公式: 

```java
prefix[i + 1] = prefix[i] + nums[i];
sum(l, r) = prefix[r + 1] - prefix[l];
```

## 2. 模板: Prefix Sum + Hash Map

适用场景: 需要判断 "之前是否出现过某个前缀状态".

| 题目                           | 核心思路                                                      |
| ---------------------------- | --------------------------------------------------------- |
| Subarray Sum Equals K        | 当前 `prefixSum - k` 以前出现过几次                                |
| Subarray Sums Divisible by K | 当前 `remainder` 以前出现过几次；相同余数之间的子数组和可以被 `k` 整除              |
| Continuous Subarray Sum      | 相同 `remainder` + 下标距离至少为 `2`；`map` 存 `remainder` 第一次出现的位置 |
| Contiguous Array             | 把 `0` 转成 `-1`, `1` 保持 `1`；相同 `prefixSum` 之间 `0` 和 `1` 数量相同 |

核心思想: 

```java
// 看当前状态需要匹配以前的哪个状态
target = currentState - k;

// 用 HashMap 记住历史状态
map.put(state, countOrIndex);
```

## 3. 模板: 1D Difference Array

适用场景: 频繁对一维数组做区间更新, 最后再统一还原数组.

| 题目                      | 核心思路                                                        |
| ----------------------- | ----------------------------------------------------------- |
| Range Addition          | `diff[l] += val`, `diff[r + 1] -= val`, 最后做一次 prefix sum 还原数组 |
| Maximum Population Year | `birth + 1`, `death - 1`, 最后做 prefix sum 找人口最多的年份             |

核心公式: 

```java
diff[l] += val;

if (r + 1 < n) {
    diff[r + 1] -= val;
}
```

最后还原: 

```java
for (int i = 1; i < n; i++) {
    diff[i] += diff[i - 1];
}
```

## 4. 模板: 2D Prefix Sum

适用场景: 频繁查询二维矩阵中某个矩形区域的和.

| 题目                             | 核心思路                                             |
| ------------------------------ | ------------------------------------------------ |
| Range Sum Query 2D - Immutable | 大矩形 - 上面 - 左边 + 左上                               |
| Matrix Block Sum               | 每个点都查一次周围 block 的矩形和, 本质是多次使用 2D prefix sum 查询矩形和 |

核心公式: 

```java
sumRegion(row1, col1, row2, col2)
= prefix[row2 + 1][col2 + 1]
- prefix[row1][col2 + 1]
- prefix[row2 + 1][col1]
+ prefix[row1][col1];
```

记忆方式: 

```text
目标矩形 = 大矩形 - 上面多余部分 - 左边多余部分 + 左上被减重的部分
```

## 5. 模板: 2D Difference Array

适用场景: 
频繁对二维矩阵中的矩形区域做更新, 最后统一还原矩阵.

| 题目                           | 核心思路                         |
| ---------------------------- | ---------------------------- |
| Increment Submatrices by One | 四个角打标记, 最后做二维 prefix sum 还原矩阵 |

核心公式: 

```java
diff[r1][c1] += 1;
diff[r2 + 1][c1] -= 1;
diff[r1][c2 + 1] -= 1;
diff[r2 + 1][c2 + 1] += 1;
```

还原时做二维前缀和: 

```java
matrix[i][j] =
    diff[i][j]
    + top
    + left
    - topLeft;
```

# 涉及的底层思维

## 1. 区间窗口思维: Prefix Sum

核心问题: 

> 区间和能不能用两个前缀状态相减得到？

典型表达: 

```text
sum(l, r) = prefix[r + 1] - prefix[l]
```


---

## 2. 区间窗口思维: Difference Array

核心问题: 

> 区间更新能不能只记录开始和结束的位置？

典型表达: 

```text
区间 [l, r] 加 val
=> diff[l] += val
=> diff[r + 1] -= val
```


---

## 3. 记忆历史思维: Hash Map

核心问题: 

> 当前状态需要和以前的哪个状态配对？

典型表达: 

```text
当前 prefixSum - 目标值 k
当前 remainder 是否以前出现过
当前 prefixSum 是否以前出现过
```


---

## 4. 扫描状态思维

核心问题: 

> 从左到右扫描时, 我需要维护什么状态？

典型状态: 

```text
prefixSum
leftSum
remainder
diff running total
map 中的历史状态
```
